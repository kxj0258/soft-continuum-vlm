# Feagine Pick-Place Tasks

## State machine

M6b implements:

```text
approach
-> align_grasper
-> close_gripper
-> lift
-> transport
-> align_place
-> release
-> retract
-> complete
```

The task never treats a close command as proof of grasp. It requires the object
state to report `grasped=true`. Lift is measured relative to the object height
recorded at reset. Placement requires the object inside the target tolerance,
the gripper open, and the object no longer grasped. Final success is delayed
until the tip retracts while the placement condition remains valid.

## Variants

- `feagine_pick_left_place_right`
- `feagine_pick_right_place_left`
- `feagine_pick_shelf_place_shelf`

All variants use the same task interface, metrics, 4D action adapter, and
deterministic expert.

## Hidden orientation control

Task context supplies the object principal axis during grasp alignment and the
place principal axis during placement. `grasper_rotation` remains absent from
the expert/policy action and is generated by `FeagineActionAdapter`.

## Metrics

Each step reports tip-object distance, object-goal distance, object lift,
grasp/lift/place success flags, and current phase index. Flags are latched for
episode diagnostics, while final success still checks the current released
placement state.

## Expert

`FeaginePickPlaceExpert` emits only `[dx, dy, dz, gripper_control]`. It uses
stage-dependent Cartesian targets for approach, lift, transport, placement, and
retraction. Alignment stages hold Cartesian position while the adapter performs
the deterministic rotation.

## Integration limitation

These tasks consume object pose, `grasped`, and robot state fields but do not
create shelf geometry or implement grasp physics. Scene naming, collision
geometry, friction, grasp attribution, and actual lift/place stability remain
real MuJoCo integration work.
